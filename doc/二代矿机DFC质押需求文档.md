# 二代矿机 DFC 质押需求文档

## 1. 背景

现有二代矿机质押只支持 OORT 质押和 OORT 产出。本次新增 DFC 质押产品，复用现有二代矿机质押入口，但质押资产、产出资产、库存、释放周期和到期退本规则与 OORT 不同。

## 2. 产品规则

DFC 质押为静态矿机产品，用户按份购买。

| 项目 | 规则 |
| --- | --- |
| 质押币种 | DFC |
| 扣款资产 | 可用 DFC |
| 产出资产 | 产出 DFC |
| 每份质押数量 | 10000 DFC |
| 每份每日产出 | 默认 27.39 产出 DFC，后台可修改，只对新订单生效 |
| 订单有效期 | 365 天 |
| 立即释放 | 当日产出的 25%，比例固定 |
| 线性释放 | 当日产出的 75%，按 365 天线性释放；比例固定，线性释放天数按订单配置/默认值 |
| 库存 | DFC 产品有库存份数；库存质押完后不能继续质押；库存后台可修改 |
| 到期处理 | 到期当天先产出最后一天收益，再将质押本金退回可用 DFC |

## 3. App 端需求

复用现有 `/api/stake/*` 接口，不新增一套 DFC 接口。

产品信息接口支持按币种查询：

```http
GET /api/stake/stakeInfo?coinType=2
```

其中 `coinType=2` 表示 DFC，`coinType=3` 表示 OORT；不传时默认 OORT，兼容旧 App。

创建质押订单仍使用现有接口：

```http
POST /api/stake/stakeOrder
```

请求中的 `id` 必须表示质押产品 ID。后端按产品的 `coinType` 判断扣款资产：DFC 产品扣可用 DFC，OORT 产品保持原逻辑扣可用 OORT。

我的质押、质押列表、线性释放列表支持可选 `coinType` 参数；不传时默认 OORT。

## 4. 后台需求

质押产品管理需要支持配置和展示：

- 质押币种
- 产出币种
- 可用库存
- 每份质押数量
- 额外质押 USDT 等值金额
- 每天产出
- 线性释放天数
- 订单有效期
- 上架状态

质押订单管理需要支持展示：

- 产品 ID
- 质押币种
- 产出币种
- 质押本金数量
- 线性释放天数
- 本金退还状态
- 本金退还时间

## 5. 兼容规则

- OORT 产品默认 `coinType=3`，`rewardCoinType=3`，线性释放天数默认 270。
- DFC 产品默认 `coinType=2`，`rewardCoinType=5`，线性释放天数默认 365。
- OORT 不校验库存；DFC 下单必须校验并扣减库存。
- DFC 不使用 OORT 的额外 120U 等值授权/扣款逻辑。
- 旧 App 不传 `coinType` 时仍查询和展示 OORT。
