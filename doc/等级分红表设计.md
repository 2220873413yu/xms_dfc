## 等级分红配置需求说明

### 1. 表用途

- `t_user_level_config` 用于配置各等级（V0、V1、V2...）的**小区业绩要求**、**是否二选一规则**以及**分成比例**。
- 后台通过维护这张表，前端根据配置展示等级列表和编辑弹窗，业务通过它判断用户是否满足升级条件。

---

### 2. 关键字段含义

- `level`：等级编码，0 对应 V0，1 对应 V1，依次类推。
- `umbrella_performance`：小区业绩门槛，单位与业绩统计保持一致（如 U 或 T），用户小区业绩达到该值视为满足业绩条件。
- `reward_ratio`：该等级对应的分成比例（百分比），用于动态分红。
- `is_two_choose_one`：是否【小区业绩 / 卡片等级】二选一：
  - 0：**否**，只考核小区业绩；
  - 1：**是**，小区业绩与卡片等级满足任意一项即可。
- `card_type`：卡片类型，仅在 `is_two_choose_one = 1` 时生效；为 0 表示不要求卡片类型，其他枚举值与 U 卡配置保持一致。

---

### 3. 升级判断规则（业务逻辑）

1. 读取用户当前等级目标配置行 `cfg`（按 `level` 匹配）。
2. 计算用户当前小区业绩 `userUmbrellaPerformance`，以及用户当前最高卡片类型 `userCardType`。
3. 根据 `cfg.is_two_choose_one` 分支：

   - 若 `is_two_choose_one = 0`（只看小区业绩）：
     - 升级条件：`userUmbrellaPerformance >= cfg.umbrella_performance`。

   - 若 `is_two_choose_one = 1`（二选一）：
     - 升级条件为**任意一项成立即可**：
       - `userUmbrellaPerformance >= cfg.umbrella_performance`
       - 或 `userCardType >= cfg.card_type`（卡片等级达到或超过配置的类型）。

4. 条件满足时，将用户等级提升到对应 `level`，并按 `reward_ratio` 参与对应等级的分红。

---

### 4. 前端展示建议（示例）

- 列表页字段：等级、**小区业绩**、**分成比例**、**是否二选一**、**卡片类型**。
- 编辑弹窗逻辑：
  - 字段「是否二选一」：下拉框（否 / 是）。
  - 当选「否」时：只展示并编辑「小区业绩」；
  - 当选「是」时：展示「小区业绩」+「卡片类型」，并在文案中说明“满足任一条件即可”。
